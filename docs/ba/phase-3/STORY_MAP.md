## Journey 1 — Onboarding / Bootstrap
- detect project context
- build project brief/context bundle
- show current state summary
- expose next recommended actions

## Journey 2 — Ask / Query / Explain
- query model context
- inspect elements/views/sheets
- explain evidence-backed findings
- keep answers grounded in current artifacts

## Journey 3 — Review / Audit / QC
- launch review lane
- gather findings
- inspect evidence
- decide fix/defer/assign

## Journey 4 — Preview / Approve / Execute
- pick bounded action
- preview delta
- confirm or reject
- verify outcome and save evidence

## Journey 5 — Connect / Report / Follow-up
- summarize findings
- prepare pilot/report artifact
- hand off or defer connector work

## Slice mapping
- **Slice 1:** Journey 1 + key Journey 2 read-only flows
- **Slice 2:** Journey 3
- **Slice 3:** Journey 4 with low-risk actions only
