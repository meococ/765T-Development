## Core questions

1. Walk me through how you currently understand a new or inherited Revit project.
2. What review/QC work is most repetitive before issue or coordination?
3. Which actions would you trust AI to preview and execute? Which would you never trust?
4. What is faster today: manual work, scripts, or existing tools like pyRevit/Dynamo?
5. If this product existed today, what would make you use it weekly?
6. What would make you refuse to use it on a real project?
7. How do approvals and accountability work in your team today?
8. What evidence or reporting would a manager need before trusting the tool?
